# Assess — The Gate Workflow

Running the gate produces two outputs: a **VERDICT** (current peg state + risk narrative) and a **RISK-PARAMS block** (pasteable config for lending protocol, listing logic, or routing guard). The workflow is always the same five steps.

---

## Step 1: Classify the Asset

Determine the asset class from the mint address or token name. Check `peg-states.md` for the 8 classes. If unknown, inspect the token's program (SPL Token, SPL Token 2022), check Helius DAS for `token_info`, and look up the issuance mechanism.

Questions that determine class:
- Is it pegged to $1 with fiat backing? → `stable_fiat`
- Is it pegged to $1 via collateral ratio and stability pool? → `stable_cdp`
- Is it an LST (stake-pool-backed)? → `lst`
- Does it accrue yield with a per-share NAV? → `stable_yield`
- Is it pegged to a non-USD fiat? → `stable_fx`
- Is it a leveraged synthetic tracking an underlying? → `synth_lev`

If you cannot determine the class, emit UNKNOWN and stop. Do not guess.

---

## Step 2: Pull Intrinsic and Market

Follow the per-class recipe in `computing-spread.md`.

Key checkpoints:
- Is the intrinsic source within its staleness bound? If not → UNKNOWN.
- Is the intrinsic non-zero? If zero → UNKNOWN (FM-5).
- Is the market quote non-null? If null (thin mint, no route) → UNKNOWN.

For Tier-2 assets (hyUSD CR, xSOL NAV): if the on-chain accounts are not decoded, print "intrinsic source not bundled — see computing-spread.md" and UNKNOWN; do not fabricate a number.

---

## Step 3: Compute Discount and Run Guards

```
discount = 1 - market / intrinsic
```

Then in order:
1. **Plausibility**: if `|discount| ≥ 1.0`, the market quote is degenerate ($0 or 2× peg). Skip this tick → UNKNOWN. Do not EWMA-update. (FM-4)
2. **EWMA** (if you have prior state): `smoothed = 0.4 × discount + 0.6 × prev_smoothed`. On first observation, use raw discount.
3. **NAV-sanity**: if direction-sensitive class AND `discount < 0` AND `|discount| × 10000 > 1000` → UNKNOWN (broken intrinsic — FM-5).
4. **Direction-aware classification**: if direction-sensitive AND `discount < 0` → PEGGED. Else classify by `|discount| × 10000` against thresholds:
   - For `stable_cdp`: run both spread-path and CR-path; take the more severe.
5. **Hysteresis** (if you have prior state): if new state is less strict than prior, apply deadband check before confirming the downgrade.

---

## Step 4: Emit VERDICT

```
VERDICT: <STATE>
Asset:   <name> (<mint>)
Class:   <asset_class>
Discount: <bps> bps (<direction>)
Intrinsic: <value> (source: <source>, staleness: <s>s)
Market:    <value> (source: Jupiter/Helius, blockId: <id>)
Reason:  <one sentence>
```

Example for a healthy jitoSOL:
```
VERDICT: PEGGED
Asset:   jitoSOL (J1toso1uCk3RLmjorhTtrVwY9HJ7X8V9yYac6Y7kGCPn)
Class:   lst
Discount: −79 bps (premium → normalizes to PEGGED — direction-sensitive)
Intrinsic: 1.1095 SOL (Sanctum, staleness: 4s)
Market:    1.1183 SOL (Jupiter quote, 1000 jitoSOL probe)
Reason:  Market is 79 bps above the redemption rate; an LST premium is demand pressure, not stress, so direction-sensitivity normalizes it to PEGGED (a symmetric check would false-alarm DRIFT).
```

Example for a stressed asset:
```
VERDICT: DEPEG
Asset:   hyUSD (HUSDm9cvmSEMBbMHpFbJwsLGKBFnM6JNXR2NHHQ7kNFi)
Class:   stable_cdp
Discount: 87 bps (discount)
CR:      126% (DEPEG — below 130% threshold)
Intrinsic: $1.00
Market:    $0.9913 (Jupiter)
Reason:  CR below depeg threshold (130%) AND market discount at 87 bps; both paths signal DEPEG.
```

## Step 5: Emit RISK-PARAMS Block

The RISK-PARAMS block is the deliverable — pasteable config for the integrating protocol.

```yaml
# RISK-PARAMS — generated by peg-risk assessment
# Asset: <name> (<mint>)
# Assessment date: <ISO date>
# Methodology: github.com/lrafasouza/pegana-replay (MIT)

asset_class:        <class>
peg_anchor:         <USD|FX|NAV|SOL>
direction_sensitive: <true|false>

# Spread thresholds (basis points)
warn_bps:           <drift threshold>      # emit monitoring alert
refuse_bps:         <depeg threshold>      # refuse new collateral / routes
liquidate_bps:      <critical threshold>   # trigger liquidation / emergency exit

# CR thresholds (if stable_cdp, in percent)
cr_warn_pct:        <drift_cr>
cr_refuse_pct:      <depeg_cr>
cr_liquidate_pct:   <critical_cr>

# Oracle staleness
oracle_max_staleness_s: <seconds>
intrinsic_source:   <pyth_feed_id|sanctum|on-chain-recipe>
market_source:      <jupiter|helius-das>

# Current state at assessment time
current_state:      <PEGGED|DRIFT|DEPEG|CRITICAL|BLACK_SWAN|UNKNOWN>
current_discount_bps: <n>
assessment_confidence: <high|medium|low — based on data source quality>
```

**Worked example — jitoSOL (Tier 1, full adapter)**:

```yaml
# RISK-PARAMS — jitoSOL
# Assessment date: 2026-06-21
# Methodology: github.com/lrafasouza/pegana-replay (MIT)

asset_class:         lst
peg_anchor:          SOL
direction_sensitive: true

warn_bps:            20
refuse_bps:          80
liquidate_bps:       250

oracle_max_staleness_s: 300
intrinsic_source:    sanctum-sol-value
market_source:       jupiter-lst-to-sol-quote

current_state:       PEGGED
current_discount_bps: -112   # premium (negative = market above intrinsic)
assessment_confidence: high
```

**Worked example — hyUSD (Tier 2 recipe, CR path)**:

```yaml
# RISK-PARAMS — hyUSD (partial — CR path requires Hylo on-chain adapter)
# Assessment date: 2026-06-21

asset_class:         stable_cdp
peg_anchor:          USD
direction_sensitive: false

warn_bps:            10
refuse_bps:          30
liquidate_bps:       100

cr_warn_pct:         150
cr_refuse_pct:       130
cr_liquidate_pct:    110

oracle_max_staleness_s: 60
intrinsic_source:    fixed-usd-1.0
market_source:       jupiter-usd-price

# NOTE: CR path requires Hylo on-chain adapter — see computing-spread.md
# Without CR data, current_state reflects spread path only
current_state:       PEGGED   # spread only; CR: UNKNOWN (adapter not implemented)
assessment_confidence: medium  # degraded: CR path missing
```

---

## Refusing to Assess

Output a blocking UNKNOWN (not a risk-params block) when:
- Asset class cannot be determined.
- Intrinsic source is unavailable, zero, or a Tier-2 class without an implemented adapter.
- Market quote is null (thin mint, no Jupiter route).
- Both sides are stale beyond their respective bounds.

Do not fabricate numbers. Do not substitute a price you found somewhere else without documenting it as the source. An assessment with missing data that produces a confident number is exactly FM-5.

---

## Notes on Confidence

`assessment_confidence` reflects data source quality:
- **high**: Tier 1 adapter; both intrinsic and market from known-good sources; staleness within tight bounds.
- **medium**: One source is near its staleness bound, or market source is Helius DAS (less depth-aware than a Jupiter route probe), or spread path only for a CDP stable.
- **low**: Source is a fallback (DexScreener, secondary venue); staleness is within bound but near limit; known-thin market.
